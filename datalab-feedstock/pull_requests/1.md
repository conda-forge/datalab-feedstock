# Align conda package requirements with upstream Python package

This PR aligns the conda package requirements with the upstream Python package as defined in the DataLab [pyproject.toml](https://github.com/DataLab-Platform/DataLab/blob/main/pyproject.toml).

## Changes

### Removed
- **opencv >=4.5** - This is an optional dependency upstream, not required. Users who need OpenCV functionality can install it separately.

### Added
- **numpy >=1.22,<2.5** - Was implicit before, now explicit to match upstream
- **packaging >=21.3** - Missing dependency that's required upstream

### Updated
- **guidata** from >=3.13.3 to **>=3.13.4**
- **sigima** from >=1.0.2 to **>=1.0.6**
- **scipy** from >=1.10.1 to **>=1.10.1,<1.17** (added upper bound)
- **scikit-image** from >=0.19.2 to **>=0.19.2,<0.27** (added upper bound)
- **pandas** from >=1.4 to **>=1.4,<3.0** (added upper bound)
- **pywavelets** from >=1.2 to **>=1.2,<2.0** (added upper bound)

### Build
- Incremented build number from **0** to **1**

## References
- Upstream dependencies: https://github.com/DataLab-Platform/DataLab/blob/main/pyproject.toml
- OpenCV is listed under `[project.optional-dependencies]`, not core dependencies

Fixes the issue where OpenCV was unnecessarily required for all DataLab installations.
