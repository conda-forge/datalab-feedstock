## Summary
This PR aligns the conda package requirements with the upstream Python package requirements from [DataLab's pyproject.toml](https://github.com/DataLab-Platform/DataLab/blob/main/pyproject.toml).

## Changes
- ❌ **Remove** `opencv >=4.5` from required dependencies (it's an optional dependency upstream under `[project.optional-dependencies]`)
- ✅ **Add** `numpy >=1.22,<2.5` explicitly (currently implicit)
- ✅ **Add** missing `packaging >=21.3` dependency
- 🔄 **Update** `guidata` from `>=3.13.3` to `>=3.13.4`
- 🔄 **Update** `sigima` from `>=1.0.2` to `>=1.0.6`
- 🔄 **Add upper bounds** to match upstream:
  - `scipy >=1.10.1,<1.17`
  - `scikit-image >=0.19.2,<0.27`
  - `pandas >=1.4,<3.0`
  - `pywavelets >=1.2,<2.0`
- 🔢 **Increment** build number from `0` to `1`

## Rationale
According to the [upstream pyproject.toml](https://github.com/DataLab-Platform/DataLab/blob/main/pyproject.toml), OpenCV is defined as an optional dependency, not a required one:

```toml
[project.optional-dependencies]
opencv = ["opencv-python-headless >= 4.8.1.78"]
```

This means users who install DataLab via conda are currently forced to install OpenCV even though it's not required for core functionality.
